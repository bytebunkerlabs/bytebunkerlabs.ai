The blind spot

No single tool sees all three
planes of AI agent activity

AI agents reach across external APIs, internal networks, and local endpoints. SASE sees the first. MCP gateways see part of the second. EDR sees a sliver of the third. Nobody sees all three — and that gap is exactly where shadow agents live.

PLANE 01

External API plane

Agent calls to model providers — OpenAI, Anthropic, Google, Bedrock. Your SASE platform sees opaque TLS, not the prompts, tool calls, or decisions inside.

SASE sees the connection, not the conversation
PLANE 02

Internal network plane

MCP servers reaching databases, Active Directory, Splunk. This traffic is deliberately bypassed from inspection to avoid latency — so it's completely unmonitored.

Most sensitive AI traffic, zero visibility
PLANE 03

Endpoint plane

Process-to-process stdio, config files, agent instructions, skills and hooks — entirely local to the workstation. Invisible to every network-based tool.

Where shadow agents are born and hide

Deploy

Runs in your cloud.
Your data never leaves.

Spectis ships as a single-tenant stack you launch into your own AWS account — one CloudFormation template, persistent storage, auto-HTTPS, secrets in your own KMS. Single-tenant by construction: your telemetry is yours alone.

Bring your own cloud, in minutes

One ARM instance, encrypted storage, Caddy auto-TLS, secrets in SSM, shell over Session Manager — no open SSH, no shared infrastructure, no data exfiltration to a vendor SaaS.

  • Single-tenant isolation — deployed in the customer's account; data stays put for as long as you want it.
  • Hardened by default — fail-closed config, TLS 1.3, security headers, least-privilege IAM.
  • Lowest-cost footprint — no ALB, no managed DB, no NAT gateway. Bring it up and down on demand.

Security posture

Built to the standard it
holds others to

A security product has to be the most secure thing in the room. Spectis is engineered that way — minimal data, modern crypto, zero interception.

Observe, never intercept

Zero inline latency, no conflict with SASE or proxy infrastructure. Telemetry only — never a man-in-the-middle.

Data minimization

Env var names only, never values. No raw command lines. No file contents unless explicitly flagged.

Zero-trust agent comms

mTLS 1.3, Ed25519-signed commands with replay protection, and short-lived agent identities. Argon2id at rest.

Auditable by design

Every action is attributed to both the agent and the human who triggered it — inspectable verdicts, a full audit trail, and evidence behind every alert.
